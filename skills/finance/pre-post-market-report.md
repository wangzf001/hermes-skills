# 盘前 · 盘后报告生成

## 触发条件
- "盘前报告" / "pre-market" / "盘前"
- "盘后报告" / "post-market" / "盘后" / "复盘"
- Cron 自动触发（盘前 7:30 / 盘后 17:00）

## 数据源

使用 `a-share-market-data` 技能拉取实时数据：
- 四大指数 + 科创50：新浪 hq API（`sh000001,sz399001,sz399006,sh000688`）
- 板块排名：Eastmoney sector list（`m:90+t2`）
- 个股行情：新浪 hq 或 Eastmoney push2
- 资讯：Eastmoney 快讯 API

# 【模式一】盘前报告

## 设计理念
每句话都给结论，每个判断都有数据，每个方向都配标的池，每个风险都有应对。

## 执行步骤

### Step 1：指数扫描
拉取四大指数 + 科创50 实时数据（前一交易日收盘或盘前竞价）

### Step 2：核心持仓扫描
拉取用户持仓（从 memory 读取最新持仓），标注涨跌和信号

### Step 3：资讯提炼
从 Eastmoney 快讯 API 抓取最近资讯，筛选 Top 5-8 条最有影响力的。

### Step 4：核心矛盾提炼
从数据和资讯中找出当日最核心的 1-2 个矛盾/方向，一句话定性。

### Step 5：策略动作
- 开盘观察信号（2-3 个）
- 今日操作建议（不动/调仓/关注）
- 回避方向

### Step 6：风险提醒
1-2 个当日最需警惕的风险

# 【模式二】盘后报告

## 设计理念
复盘不是记流水账，是迭代判断框架。盘前说什么→实际发生什么→对为什么对，错为什么错→明天怎么调。

## 执行步骤

### Step 1：盘面数据
四大指数涨跌 + 日内走势特征 + 成交额 + 涨跌家数

### Step 2：板块复盘
最强 1-2 个板块 + 最弱 1 个板块，三段式拆解：
- 涨跌驱动
- 盘前预判验证（✅/❌/🔄）
- 持续性重新评估

### Step 3：持仓复盘
逐只扫描：涨跌 + 与盘前预判对比 + 信号变化

### Step 4：盘前预判逐条验证
从盘前报告中提取核心判断，逐条验证 → 形成迭代闭环

### Step 5：明日关注框架
- 核心观察变量
- 值得留意方向
- 警惕风险
- 仓位建议

## 迭代闭环

```
盘前：给出判断
  ↓
盘中：市场实际运行
  ↓
盘后：验证判断（✅❌🔄 + 迭代教训）
  ↓
下一次盘前：调用修正后的框架
```

## 存储
- 盘前报告 → `报告文件/YYYYMMDD-盘前报告.md`
- 盘后报告 → `报告文件/YYYYMMDD-盘后报告.md`

## 合规
- 不推荐具体买卖价位
- 每篇末尾强制免责声明
- 禁"买入/卖出/必涨/翻倍/梭哈/目标价"
