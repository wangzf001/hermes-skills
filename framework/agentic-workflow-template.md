# Agentic 工作流模板 v1.1

给 AI 看的团队搭建说明书。任何领域都能用。

## 核心理念
你不是工具，你是团队。每个 Skill = 一个独立角色。

## 四层闭环架构

```
采集层(并行) → 分析层(并行) → 决策层(串行) → 🆕反馈层(事后)
```

## 🆕 三档运行模式

| 模式 | 触发 | 出动 | 耗时 |
|------|------|------|:--:|
| ⚡ 预警 | 定时/阈值 | 1采集+风控 | <5s |
| 🚀 快速 | 扫盘/轻量查询 | 2采集+策略师 | ~15s |
| 🏛️ 完整 | 深度/月审 | 全员四层 | ~60s |

## 🆕 反馈层 · 能力进化引擎

**复盘师**：回捞预判→对照结果→逐条标记(✅❌🔄)→归因→提炼迭代日志→下次自动加载

**绩效追踪师**：每个角色拥有持久「能力卡」
- 胜率/偏差方向/最强维度/最弱维度/近10次趋势
- 跨会话保存，越用越准

**权重自适应**：策略师不再等权平均三份报告
- 根据近10次胜率动态调权
- 胜率差<10%等权 | 10-20%±0.05 | >20%±0.10
- 上限0.50 | 样本<15次不参与加权

> 第一周凭直觉管团队，三个月后数据告诉你该听谁。

## 🆕 分歧裁决 v1.3

### 四类分歧
| 类型 | 表现 | 策略 |
|------|------|------|
| 🔴 方向分歧 | 结论相反 | HITL 升级请示 |
| 🟡 幅度分歧 | 方向同力度异 | 置信度加权 × 历史胜率 |
| 🟡 归因分歧 | 事实同原因异 | 综合呈现，不裁决 |
| 🟢 置信度分歧 | 结论同信心异 | 取高置信度方 |

### 强制置信度
每份报告必须输出：结论 + 置信度(1-5) + 依据 + 不确定因素。空口喊信心不通过质检。

### 加权公式（幅度分歧专用）
最终权重 = 置信度(40%) + 历史胜率(40%) + 近10次趋势(20%)

### 历史模式匹配
复盘师记录每次分歧 → 下次类似分歧自动提示"上次XX对了"

### 升级规则
方向分歧 / 置信度差>2 / 胜率差>20% → 任一触发即 HITL

## 🆕 应急降级
API全挂 → 离线模式：用缓存+标注不可得

## 搭建流程
1. 拆领域 → 2. 定角色(每层2-3人) → 3. 写Skill → 4. 设三档触发 → 5. 加复盘师 → 6. 配cron

## 14项检查
采集≤3人 | 分析独立方法论 | 有核查 | 有质检 | 最大限度并行 | HITL清晰 | Reflexion≤3轮 | 一角色一Skill | 数据缓存 | 执行日志 | 🆕有复盘师 | 🆕三档模式 | 🆕分歧裁决 | 🆕降级路径
